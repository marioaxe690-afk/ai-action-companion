# Project Context

## 产品名

刷到即行动

## 一句话定位

把用户刷到、看到、想到的内容，转化为马上能执行的下一步行动。

## 当前阶段

Vite + React + TypeScript 前端 mock 原型。当前版本是 Prototype v0.2，不接真实 AI 服务，不包含后端、数据库、登录注册或真实内容识别。

当前原型采用暗色 AI 玻璃工作台风格，用于强化黑客松展示感和 AI 产品气质。

## 核心流程

1. 用户输入短视频文案、截图通知、学习资料、比赛信息、项目灵感或生活目标。
2. 系统展示 mock AI 场景理解，包括内容类型、用户意图、推荐转化、紧急程度、风险判断和适合赛道。
3. 系统生成行动卡片、任务链和下一步计划。
4. 用户勾选行动卡后，页面同步更新完成进度。
5. 用户切换陪伴人设，同一行动卡获得不同语气的行动反馈。
6. 页面展示 mock 用户行动画像，表达“下次更懂你”的轻量记忆方向。

## 第一版深做场景

第一版深做“比赛通知”，以 AI 黑客松比赛通知为主示例。重点演示报名、组队、定方向、准备作品、路演表达这条链路，避免过早扩展到真实截图识别、视频抓取或外部平台数据。

## 不做事项

- 不接真实 AI API
- 不添加后端
- 不添加数据库
- 不添加登录注册
- 不做图片真实识别
- 不做视频抓取
- 不做抖音数据抓取
- 不做浏览器插件
- 不引入复杂依赖

## 安全红线

- 不写任何 API Key、Token、secret。
- 不采集真实用户账号、 Cookie、浏览记录或平台数据。
- 不声称已经完成真实 AI 识别、OCR、视频理解或抖音数据抓取。
- 当前所有 AI 理解、行动卡和记忆画像都必须标注为 mock 数据或前端原型能力。

## 下一步计划

- 为更多输入类型补充 mock 场景理解和行动卡模板。
- 细化行动卡状态，支持完成、卡住、跳过和需要重拆。
- 准备黑客松路演脚本，突出“内容行动化”与普通聊天 AI、普通 Todo 的差异。
- 在考虑真实 AI 服务前，先完成隐私、数据留存、提示词安全和用户授权设计。
